# Project Workflow - Insurance Fraud Detection

## Step 1: Problem Definition
- Understand the problem of insurance fraud
- Define objectives and success metrics

## Step 2: Data Collection
- Collect dataset from Kaggle (Auto Insurance Claims Data)
- Understand the features and target variable

## Step 3: Data Preprocessing
- Handle missing values
- Encode categorical variables
- Scale numerical features
- Handle imbalanced data using SMOTE

## Step 4: Exploratory Data Analysis (EDA)
- Visualize data distributions
- Find correlations between features
- Identify patterns in fraudulent claims

## Step 5: Model Building
- Split data into train and test sets
- Train models: Logistic Regression, Decision Tree, Random Forest, XGBoost
- Compare model performances

## Step 6: Performance Testing & Hyperparameter Tuning
- Evaluate using Accuracy, Precision, Recall, F1-Score, ROC-AUC
- Tune hyperparameters using GridSearchCV

## Step 7: Model Deployment
- Build a Flask web app
- Deploy the model for real-time fraud prediction

## Step 8: Documentation & Demonstration
- Write project report
- Create presentation
- Demo the working model
